8 этапов производства

Полный маршрут от идеи до готового ролика. Жми на этап — раскроется.

1 · Бриф — что собрать ДО старта
фото-якоря и вводные, без которых посыплется сходство
30–60 мин 0 кредитов Готовый пакет для старта

Фото человека для аватара:

  • 10–15 фото — ровный нейтральный свет, без сильного макияжа и фильтров
  • Ракурсы: фас, оба строгих профиля (90°), ¾ слева и справа — все 5 обязательны
  • Планы: крупно лицо + по пояс; 2–3 эмоции (нейтраль, лёгкая улыбка, серьёзный взгляд)
  • Одинаковый свет на всех фото = стабильнее сходство при обучении аватара

Вводные проекта: формат (16:9 / 9:16), хронометраж, стиль (палитра/свет) + 2–3 референса, сквозные правила — что нельзя менять между сценами (цвет акцента, элемент одежды, цвет фона).

Именуй файлы по ракурсу — face_front.jpg, face_left90.jpg — штаб не путается в папке.

Pro: сними 2–3 фото только для сложного ракурса (строгий профиль 90°) на нейтральном фоне — именно он чаще всего теряет сходство при генерации.

Не стартуй без полного набора ракурсов — модель додумывает лицо там, где фото не хватает. Особенно критичен профиль.

Готово, когда: есть папка с фото по ракурсам, список сцен и сквозные правила — штаб ничего не додумывает.

2 · Сюжет и ТЗ — утверждаем ДО производства
закрыть вопросы на бумаге, пока правки бесплатны
1–2 часа 0 кредитов Подписанное ТЗ = 0 кругов правок

Что входит в ТЗ заказчику:

  • Концепт и стиль (1–2 предложения + 2–3 визуальных референса)
  • Раскадровка: Сцена 3 — герой у окна, ¾ слева, лёгкий поворот головы, 4с
  • Лук: одежда, причёска, аксессуары, акцентный цвет — одинаковы во всех сценах
  • Что критично сохранить (идентичность) и что нельзя менять (фоновый цвет, элемент бренда)
  • Смета на генерацию: число сцен × кредиты заранее — без сюрпризов для клиента

Утверди письменно (скриншот чата / подпись) → правишь сцену на скетче бесплатно, не на сгенерированном видео.

Pro: посчитай смету ДО старта — 10 сцен × 4с на kling-turbo ≈ 60 кр, на seedance ≈ 200–360 кр. Заказчик одобряет бюджет на берегу.

Не начинай рендер без подписанного ТЗ. Правка на скетче — минута, на готовом видео — повторная генерация и кредиты дважды.

Готово, когда: раскадровка подписана, смета согласована, лук зафиксирован.

3 · Стек — маршрут инструментов
что где делает и где горят деньги, а где бесплатно
разово ~1 час настройки обработка бесплатно Понимаешь, где тратить, где нет

Полный маршрут: генерация в облаке → обработка локально → сборка:

Higgsfield

Платно — здесь горят кредиты

Штаб: пишет промты, управляет Higgsfield (MCP), запускает скрипты обработки, ведёт QC.
Не тратит кредиты Higgsfield

Стабилизация, грейд, 4K, склейка, скорость. Весь постпродакшн — локально.
Полностью бесплатно

Правило стека: облако — только для генерации (платно). Всё остальное — локально, бесплатно. Не гони постпродакшн через Higgsfield.

Готово, когда: понимаешь, где тратишь кредиты (только Higgsfield), а где работаешь бесплатно.

4 · Claude Code как штаб
один «мозг» держит весь проект — стиль, промты, QC
0 кредитов Higgsfield Единый стиль во всех сценах

Что делает Claude Code в этом проекте:

  • Держит правила проекта — лук, палитра, сквозные запреты — в одном файле. Все промты строятся по одним правилам → стиль не плывёт между сценами.
  • Пишет и чинит промты под каждую сцену по шаблону (ракурс, якорь, негатив, цвет-правило) — не нужно делать вручную.
  • Управляет Higgsfield через MCP — запускает генерации, получает результаты без ручных кликов в браузере.
  • Запускает постпродакшн пакетом — один вызов грейда / стабилизации / 4K через ffmpeg или OpenCV на все клипы сразу.
  • Ведёт QC-чеклист — автопроверка каждого клипа: нет фризов, сходство в норме, нет двойника в кадре.

Pro: вставь этот гайд в системный промт сессии — штаб сразу знает правила проекта, методику и стек. Работает как брифованный ассистент, не надо объяснять каждый раз.

Готово, когда: один «мозг» держит проект, между сценами ничего не плывёт, постпродакшн запускается одной командой.

5 · Стартовые кадры (дёшево)
картинка под сцену + якорь сходства до оживления
15–30 мин на сцену ~2–4 кр / кадр Утверждённый стоп-кадр перед дорогим оживлением

Слоты промта стартового кадра:

  • Субъект + якорь сходства — «The SAME woman as the reference — keep her exact face»
  • Ракурс — строгий профиль, ¾, фас, полный рост (чётко, не «примерно»)
  • Лук — одежда, детали, акцентный цвет (одинаково во всех сценах)
  • Сцена / фон / свет — где и как освещено
  • Цвет-правило — напр. «black & white EXCEPT lips in red»
  • Гард — «ONE single woman only, no duplicate, no extra hands»

Воркфлоу: генери 3–4 варианта → выбери лучший → зафиксируй seed и настройки → получи утверждение → только потом оживляй.

Якорь сходства = реальное фото или обученный аватар, а не текст. На словах модель «идеализирует»: нос тоньше, скулы острее.

Pro: держи один эталонный кадр (утверждённый лук) и сверяй каждый новый кадр с ним перед отправкой заказчику — так не пропустишь дрейф стиля.

Профиль 90° на одном тексте не делай — самый «непохожий» ракурс. Дай реальное боковое фото как жёсткий референс + запрет «не сужать, не заострять».

Готово, когда: кадр утверждён заказчиком → безопасно оживлять.

6 · Оживление — здесь горят кредиты
картинка → видео без брака и без слива бюджета
3–8 мин / клип 6–36 кр / 4с Чистый клип без фризов и деформаций

Правила генерации клипа:

  • Одно действие на клип, база 4 сек — не перегружай промт
  • Камеру фиксируй словами: static shot, no zoom, no pan — иначе модель добавит наезд
  • Мимика «very subtle» — голову не вертеть, эмоция только взглядом + подбородком
  • После генерации — QC: фризы, двойник, деформация лица
  • Всегда генери 2 дубля → бери чистый

Модель под бюджет: общий план / мелкое лицо → kling3_0_turbo (≈6 кр/4с); крупный план / важно сходство → seedance_2_0 + аватар (≈18–36 кр/4с).

Примерная смета 10 сцен × 4с: kling-turbo ≈ 60 кр · seedance ≈ 180–360 кр.

Частые ошибки → решения:

  • Деформация лица на крупном → аватар + подмена лица в nano_banana_pro + широкий план
  • «Двойник / два человека» → добавь гард «ONE person only, no duplicate», генери 2 — бери чистый
  • «Убрать предмет из руки» — не работает → бери базовый кадр, где рука уже пустая
  • Наезд камеры сам по себе → camera stays static, no movement в промте

Готово, когда: клип без фризов, без деформаций, сходство держится на всей длине.

7 · Грейд / стабилизация / 4K — бесплатно
локально, 0 кредитов Higgsfield
5–15 мин / клип (авто) 0 кредитов Кадр как со штатива, кино-палитра

Стабилизация — зачем: после оживления модель часто добавляет дрожь и наезд. Дрожь читается как «дёшево / ИИ».

  • deshake (vidstab) — убирает дрожь кадра
  • Лок размера (обратный зум) — убирает наезд → кадр стоит ровно, как со штатива
  • unsharp — лёгкая резкость после стабилизации

Грейд (цвет): Ч/Б-база + возврат цвета только в нужной зоне по маске. Лёгкое зерно (noise filter) + Ч/Б-база с возвратом цвета через маску = «киношность» без платных плагинов.

4K: scale + unsharp через ffmpeg — бесплатно. Для Instagram достаточно 1080p.

Pro: запускай стабилизацию и грейд одним скриптом на всю папку клипов — Claude Code автоматизирует пакет через ffmpeg, не надо делать вручную каждый файл.

Платный AI-апскейл не нужен для Instagram — из 1080p норм. Не трать кредиты там, где ffmpeg справляется бесплатно.

Готово, когда: кадр как со штатива, палитра ровная на всех сценах, 0 кредитов потрачено.

8 · Сборка и звук (CapCut)
ритм, монтаж и экспорт без джаддера
1–3 часа CapCut бесплатен Финальный ролик — 1 экспорт, без джаддера

Структура ролика:

  • Хук 0–3с — сильный кадр или движение с первого фрейма, держит внимание
  • Тело — жёсткие резы под бит, единый fps
  • Концовка / CTA — затемнение или стоп-кадр + призыв (если нужен)

Аудио-first: сначала кладёшь дорожку (музыка / SFX) как ритм-скелет, потом режешь клипы под бит. Бесплатные библиотеки звука: Pixabay, Freesound, YouTube Audio Library.

Монтаж: открытие из чёрного (плавное появление ~0.7с), между сценами — жёсткие резы. Оптический поток ВЫКЛ — добавляет артефакты на AI-видео.

Параметры экспорта: 1080p · 24 fps · H.264 · ~20 Мбит/с — и один раз, не пересохранять.

Pro: 24fps воспринимается как «кино». 30fps — как «обычное видео». Для брендового ролика 24fps выигрывает всегда.

Не ставь 30 fps и не пересохраняй — появится джаддер (дёрганость). Экспортируй один раз финальный файл.

Готово, когда: ритм держит внимание от первого до последнего кадра, экспорт один раз, без джаддера.

Claude Code — штаб,
который экономит кредиты

Как шла работа

— держит контекст и правила проекта в файлах, пишет промты;

— управляет Higgsfield через MCP — генерации из одного места, без ручных кликов;

— грейд / стабилизация / 4K скриптами ffmpeg / OpenCV — бесплатно;

— берёт дешёвую модель под задачу и ловит брак на проверке до повторной генерации.

Подключение и запуск (для новичка)

Если Claude Code ещё нет: скачай десктоп-приложение Claude Code (Windows/Mac) на официальном сайте Anthropic (раздел Claude Code), либо поставь CLI в терминал.

Включить коннектор Higgsfield (прямо в приложении) — вставь команду:

claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp

→ при первом вызове откроется браузер — войди в свой Higgsfield (OAuth). Проверь: /mcp.

Локальные инструменты (терминал, не MCP):

  • ffmpeg → winget install ffmpeg
  • OpenCV + Python → pip install opencv-python numpy
  • curl — встроен в Windows

Как выполнить команду: команды claude…/winget…/pip… вставляй в терминал (Windows: Пуск → набери «PowerShell» → Enter → вставь строку → Enter). Команды со слешем (/mcp) — внутри сессии Claude Code.

Про скиллы — честно: под видео отдельные скиллы не нужны, всё держится на Higgsfield MCP + локальном ffmpeg/OpenCV.

Памятка: как не сжечь
деньги и время

  • Лок лука → дешёвый кадр → утвердил → оживил
  • Реальное фото > промта на сходстве
  • Дешёвая модель на мелком лице
  • Грейд / 4K на ffmpeg, не на кредитах
  • Минимум мимики при оживлении
  • Проверка (QC) до сборки
  • Экспорт раз, 24 fps

Частые вопросы

Нужен ли дорогой ПК?

Нет — тяжёлое считают облачные модели; локально только бесплатный ffmpeg.

Сколько уходит кредитов?

Зависит от числа сцен; принцип «кадр дёшево → оживление на утверждённом» режет расход в разы.

Можно без обученного аватара?

Для общих планов — да; для крупных лиц аватар спасает сходство.

С чего начать?

С брифа (этап 1) и принципа экономии (блок «Главный принцип»).
